# Alembic Migrations

Workflow for schema changes.

## Workflow

1. Update SQLAlchemy model(s) in `api/models.py`
2. Generate migration: `uv run alembic revision --autogenerate -m "description"`
3. Review migration in `alembic/versions/`
4. Apply: `uv run alembic upgrade head`
5. Verify downgrade: `uv run alembic downgrade -1`
6. Re-apply: `uv run alembic upgrade head`
7. Tests: `uv run pytest`
8. Update docs/schema.md

## Rules

- Never edit old migrations; create corrective migration
- Always review autogenerated output
- Every migration needs downgrade() unless data loss explicitly approved
- Update docs/schema.md when schema changes

## SQLite Pitfalls

- Column rename: may emit DROP + ADD, losing data
- Non-null on existing table: backfill or set default first
- Multiple heads: resolve before continuing
